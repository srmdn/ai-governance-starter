# Model and Provider Standards

Use this checklist when selecting or changing models/providers.

## Security and Privacy

- data retention policy reviewed
- customer data handling documented
- encryption and access controls verified
- regional/data residency needs addressed

## Reliability and Quality

- latency and uptime meet requirements
- evaluation set covers realistic workloads
- hallucination/error patterns understood
- fallback strategy defined

## Safety and Compliance

- abuse and misuse controls assessed
- moderation and policy features evaluated
- licensing and ToS constraints reviewed

## Cost and Operations

- cost per request estimated
- rate limits and quotas understood
- lock-in and portability considerations documented

## Approval

For Tier 2+ workloads, record provider/model rationale in a decision record.
