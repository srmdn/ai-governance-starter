# AI Incident Response

## What Is an Incident

Any event where AI behavior causes or could cause harm, security issues,
policy breaches, or material user impact.

## Severity Levels

- Sev 1: critical safety/security or widespread harmful impact
- Sev 2: significant user impact, no immediate critical threat
- Sev 3: limited impact or near miss

## Response Process

1. Detect and report incident with timestamp and owner.
2. Contain: disable feature, rollback, or apply guardrails.
3. Triage severity and affected scope.
4. Mitigate and communicate status updates.
5. Recover with validation checks.
6. Run post-incident review with corrective actions.

## Post-Incident Record

Capture:
- root cause
- contributing factors
- missed detections
- immediate fixes
- long-term preventive actions
- policy/process updates required
