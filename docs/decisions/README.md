# Decision Records

Use this directory to store architecture and governance decisions.

## Naming Convention

`YYYY-MM-DD-short-title.md`

Example:
`2026-03-23-adopt-tiered-ai-risk-model.md`

## When to Write One

- model/provider changes for Tier 2+
- policy exceptions
- major architecture decisions affecting safety, security, or governance
- incident-driven control changes

Start from `docs/decisions/templates/decision-record-template.md`.
