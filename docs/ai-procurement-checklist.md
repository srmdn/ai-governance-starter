# AI Procurement Checklist

Use before adopting a new AI model/provider for product or internal workflows.

## 1. Business Fit

- [ ] clear use case and expected business outcome
- [ ] measurable success metric defined
- [ ] fallback option documented

## 2. Data and Privacy

- [ ] data retention terms reviewed
- [ ] training-on-your-data policy confirmed
- [ ] sensitive data classification mapped to allowed usage
- [ ] data residency requirements checked

## 3. Security and Compliance

- [ ] access controls and key management reviewed
- [ ] incident response and breach notification terms reviewed
- [ ] compliance requirements mapped (if applicable)

## 4. Reliability and Operations

- [ ] uptime/SLA expectations documented
- [ ] latency and rate limits tested against workload
- [ ] provider outage fallback strategy defined

## 5. Legal and Commercial

- [ ] ToS and indemnity terms reviewed
- [ ] IP/licensing implications reviewed
- [ ] total cost estimate documented (baseline and growth scenarios)
- [ ] lock-in risks and exit path captured

## 6. Approval

- [ ] decision record created in `docs/decisions/`
- [ ] maintainer approval recorded
